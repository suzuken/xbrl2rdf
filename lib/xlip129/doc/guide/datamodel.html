Overview

An XLink data model consists of an XLink data set and at least one DOM tree related to the XLink data set.
An XLink data set contains the results of analyzing XLink descriptions in XML documents and the relationship between XML documents and DOM trees[Steve1].
Applications can refer to information written in XLink by a linkage of DOM and an XLink data set in the configuration described above for an XLink data model.

Overview of main classes

class XLinkDataSet

This is the main unit of an XLink data set.
This is the highest class that manages the XLink information in an XLink data model. All results analyzed by XLiP can be obtained, starting from an XLink data set.

class XLinkDeclaration

This is link declaration information.
Extended-type[Steve2] elements and simple-type elements defined in XLink are modeled in this class. This class provides a starting point from which information of elements under the class is obtained.

class Arc

This is arc information.
Arc-type elements defined in XLink are modeled in this class. This class provides detailed information related to traversal.

class Locator

This is locator information.
Locator-type elements and resource-type elements defined in XLink are modeled in this class.
This class provides information on the resource locations of link start and end positions and related information.

class Address

This is address information.
href attribute values of locator-type elements defined in XLink are modeled in this class. This class provides URI and fragment identifiers included in URI.

class Title

This is title information.
Title-type elements are modeled in this class.
This is not the title attribute information defined as a global attribute of XLink.
