Application Image

Figure 1 shows an application image where XML document links are processed based on the XLink standard.




Figure 1 Application image


An application requests the XML processor and the XLink processor that generates DOM to perform processing, and it accesses the generated information space. The information space is referred to as an XLink data model.
An XLink data model consists of an XLink data set and at least one related DOM.
An XLink data set contains the results of analyzing XLink descriptions in XML documents.

Processing Flow

A model used for processing link information defined in accordance with XLink consists of the following two processes.
  1. Constructing an XLink data model
  2. Referencing an XLink data model

Constructing an XLink data model

An XLink data model consists of the following two data items.
  • DOM tree (XML documents including XLink descriptions)
  • XLink data set (results of analyzing a DOM tree)
XLiP analyzes a received DOM tree to construct an XLink data set.
The link base drawn up by XLink is also analyzed. In this case, if another XML document specified as a target of "onLoad" is found, a DOM tree corresponding to the XML document is generated and an XLink analysis is performed. This is a reflexive operation, so an application that uses XLiP need not consider it.
This operation enables XLiP to construct an XLink data model that consists of an XLink data set and multiple DOM trees.



Figure 2 Constructing an XLink data model


Referencing an XLink data model

An XLink data model is referenced from an XLink data set. For details on referencing an XLink data model from an XLink data set, see Referencing an XLink Data Model.
The XLink processor provides a means of extract information from an XLink data set.

The time of referencing an XLink data model can be determined according to the context of an application (e.g., at the request of a user). Users can use this function as follows:

  • An XML document for which XLink analysis has been performed is assumed to be displayed. When a user clicks a certain location, an XLink data set is referenced.
  • Immediate after construction of an XLink data model, the user obtains the link information set to a node in a document that starts traversal from an XLink data set, and obtains the link information the user wants.
