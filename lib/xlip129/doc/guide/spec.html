XLiP Standard Specification List

Condition of XLiP Operation

XLiP requires a library that satisfies the following standard specifications:
  • The World Wide Web Consortium (W3C) Technical Reports
    • Extensible Markup Language (XML) 1.0 (Second Edition)
    • Namespaces in XML
    • Document Object Model (DOM) Level 2 Core Specification Version 1.0
  • Java Specification Requests
    • Java™ API for XML Processing (JAXP) 1.1

Standard Specifications to Which XLiP Complies

XLiP complies to the following standard specifications:
  • W3C recommendations
    • XML Linking Language (XLink) Version 1.0
    • XML Path Language (XPath) Version 1.0
    • XML Base
  • Candidates in W3C recommendations
    • XML Pointer Language (XPointer) Version 1.0
  • Others
    • Different standard specifications to which the above standard specifications are predicted to be applicable[Steve1]: The Internet Engineering Task Force (IETF) Request for Comments, W3C Technical Reports, ISO and others

Restrictions

XLiP has the following restrictions.
  1. About fragment identifiers
    Currently, if xpointer specifies referencing part of a DOM tree, such as its subtree, the entire DOM tree including the subtree is read.
For details on other restrictions, see the API Specifications.
